AssignmentCheck: automatic check for your students

George Moroz

2017-03-14

1. Installation

Since AssignmentCheck is an R package, you should install R on your PC if you haven’t already done so. To install the AssignmentCheck package, run the following command at your R IDE, so you can also get the development version from GitHub:

install.packages("devtools")
devtools::install_github("agricolamz/AssignmentCheck")

Load package:

library(AssignmentCheck)

2. Checks

Imagine that we have a directory with some files from your students

list.files()
## [1] "test_1.Rmd" "test_2.Rmd" "test_3.Rmd" "test_4.Rmd" "test.txt"

As you can see, there are .Rmd files, and one .txt file. This files have awfull names, but believe me: even if you ask your student to name files during some pattern, you will recieve a couple files with nasty names. test_files function can take all .Rmd files and check, whether R code in R chunks produce the same result that we expected. Our expectations should be provided via list().

fit <- lm(mpg~cyl, mtcars)
test_files(expectations = list(4, fit$coefficients))
##    file_name           author ### 1.1 ### 1.2 results
## 1 test_1.Rmd       Tom Sawyer    TRUE    TRUE       2
## 2 test_2.Rmd Huckleberry Finn    TRUE    TRUE       2
## 3 test_3.Rmd       Joe Harper   FALSE    TRUE       1
## 4 test_4.Rmd     Amy Lawrence   FALSE    TRUE       1

As you can see, there are students name from name: field, and summary on each student.

There is a variable keyword_prefix that defines the common part of the keywords, so you can ask your students to put their answer code chunks after some keywords. By default keyword_prefix is "### ", so your students can write “### 1.1”, “### 1.2” and so on. But you can specify your own prefixes.
